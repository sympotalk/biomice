# BioMICE 모바일 재설계 브리프 — 클로드디자인용

> 이 문서를 통째로 클로드디자인에 붙여넣으면 됩니다.
> 출력 결과물은 `uiux/` 폴더에 JSX로 저장 → 그 시안을 보고 실제 코드로 구현.

## 0. 프로젝트 한 줄 요약

**BioMICE (바이오마이스)** — 국내 의학 학술대회 정보를 모은 플랫폼.
대한의학회(KAMS) 데이터를 매일 크롤링해서 의사·제약사 직원에게 제공하고, Featured/배너로 광고 수익을 낸다.

- **타겟 유저:** 의사 (학술대회 일정 확인·즐겨찾기), 제약사 마케터 (광고 집행)
- **운영 도메인:** https://biomice.xyz
- **레퍼런스:** [myfair.co](https://myfair.co) — 메디컬 섹션의 컴팩트한 리스트 스타일을 따라가고 싶음

## 1. 이 브리프의 목적

**현재 모바일 사이트가 완전히 깨져 있음. 모바일 우선으로 처음부터 다시 설계해줘.**

### 지금 깨진 점 (실제 모바일 스크린샷에서 확인)
1. 홈 검색바의 "검색" 버튼이 화면 오른쪽으로 잘려나감 — 컨테이너가 viewport 너비를 넘음
2. 카드 한 장이 화면 절반 이상을 차지 — 이미지 영역 160px 고정 + 큰 패딩 때문. 정보 밀도가 0에 가까움
3. myfair.co의 핵심 가치인 "한 화면에 여러 학술대회 정보를 컴팩트하게" 느낌이 전혀 안 남
4. 데스크톱 카드를 그냥 1열로 늘려놓은 모양새 — 모바일 전용 디자인이 없음

### 결과물에서 기대하는 것
- 360px(아이폰 SE) ~ 414px(아이폰 Pro Max) 폭에서 자연스럽게 보이는 디자인
- 한 화면에 학술대회 카드가 **3~5개는 보이는** 정보 밀도
- 터치 타겟 최소 44×44px
- 데스크톱(≥1024px) 디자인은 유지하되 별도 시안으로 함께 제공

## 3. 핵심 디자인 원칙 (★ 매우 중요 ★)

### 3.1 모바일 우선
- **기준 너비: 360px** (아이폰 SE / 갤럭시 S 표준)
- 어떤 컴포넌트도 360px에서 옆으로 잘리거나 가로 스크롤이 발생하면 안 됨
- 모든 패딩은 좌우 14px 기준 (= viewport 360 - 332 콘텐츠 폭)

### 3.2 정보 밀도 (myfair.co 스타일)
- 학술대회 카드는 **이미지 없는 텍스트 위주의 가로형 리스트** 권장
- 이미지(학회 로고)는 카드 왼쪽에 작게 (40~48px) — 데스크톱처럼 큰 hero 영역 금지
- 한 카드 높이는 100~120px 안쪽 → 한 화면(viewport 800px 가정)에 5~7개 보이도록

### 3.3 터치 인터랙션
- 모든 클릭 가능 영역 최소 44×44px
- 카드 전체 클릭 → 상세 페이지
- 즐겨찾기 하트는 카드 안에서 별도 터치 영역 (이벤트 전파 차단)

### 3.4 상단 고정(sticky) 활용
- 헤더는 sticky
- 학술대회 상세에서 "사전등록" CTA는 모바일에서 화면 하단 고정 (bottom sticky bar)

## 4. 페이지·컴포넌트별 설계 요구사항

### 4.1 헤더 (Header)

**모바일 (≤768px):**
- 높이 56px
- 좌측: 로고 (24px 높이)
- 우측: 햄버거 (≡) — 클릭 시 우측에서 슬라이드 인 드로어
- **검색 아이콘** 추가 (햄버거 왼쪽) — 클릭 시 풀스크린 검색 모달 (myfair 스타일)
- 드로어 내용:
  - 상단 X 버튼 + 로고
  - 메뉴: 학술대회 / 이번 주 / Featured / 학회 / 제약사
  - (로그인 시) 마이페이지 / 로그아웃
  - (비로그인) 하단 고정 로그인·회원가입 버튼

**데스크톱 (≥1024px):**
- 높이 64px
- 좌측: 로고 + 메뉴
- 우측: 검색바(인라인) + 로그인/회원가입 버튼

### 4.2 홈 — 히어로(Hero)

**모바일:**
- 패딩 36px 14px 28px
- 알약 배지 "● 대한의학회(KAMS) 데이터 연동 · 매일 자동 업데이트" — 한 줄 안에 들어가도록 폰트 11~12px
- 헤드라인 22~24px / 800 / "국내 의학 학술대회를\n한 곳에서"
- 부제 13px / "대한의학회 등록 학술대회 N건을 한눈에"
- **검색바**:
  - 컨테이너 폭 = `100% - 28px` (양쪽 14px 패딩 안에)
  - "검색" 버튼이 절대 잘리지 않게 — 검색 아이콘+input은 flex:1, 버튼은 flex-shrink:0, 고정 폭 64px 정도
  - 또는 모바일에서는 검색 버튼 제거하고 input만 (엔터로 검색)
- 진료과 칩 — 가로 스크롤로 (현재 wrap이지만 스크롤이 더 myfair스러움)

### 4.3 학술대회 카드 (가장 중요!)

**현재(잘못됨):** 데스크톱 카드 = 상단 160px 이미지 + 하단 텍스트. 모바일에서 1열로 1카드가 너무 큼.

**모바일에서 원하는 모양 (가로형 리스트):**

```
┌──────────────────────────────────────────────┐
│ ┌──┐  내과  D-19              ♡  │
│ │로│  2026 대한내과학회 춘계학술대회    │
│ │고│  📅 2026.05.14 – 05.16            │
│ └──┘  📍 그랜드 워커힐 서울 · 서울    │
└──────────────────────────────────────────────┘
```

- 폭: 100%, 높이: 약 110px
- 좌측 로고: 48×48px (로고 없으면 학회 약자)
- 우측 정보 영역:
  - 1줄: 진료과 칩(11px) + D-day 칩 + 우측 끝 즐겨찾기 하트
  - 2줄: 학술대회 제목 (14px / 600 / 1줄 ellipsis)
  - 3줄: 📅 날짜 (12px / 회색)
  - 4줄: 📍 장소 (12px / 회색 / 1줄 ellipsis)
- Featured 카드: 좌측 보더 3px 앰버, 배경 var(--bm-accent-subtle) 약간 또는 우측 상단에 작은 "🔥 Featured" 배지

**데스크톱:** 현재 형태(상단 이미지 + 하단 텍스트) 유지하되 그리드 4열

### 4.4 홈 — 섹션 구성

각 섹션 헤더: 제목 18px / 700 + 부제 12px / 회색 + 우측 "더보기 →" (12px / 메인색)

1. **광고 배너** (있을 때만, AdBanner)
2. **Featured 학술대회**
   - 데스크톱: 4열 그리드
   - 모바일: 가로 스크롤 (1.2개씩 보이게, 카드 폭 280px) — 데스크톱 카드 형태 유지
3. **이번 주 열리는 학술대회**
   - 데스크톱·모바일 모두 가로 스크롤
   - 카드 폭 데스크톱 280px, 모바일 240px
4. **다가오는 학술대회**
   - 데스크톱: 4열 그리드 (현재 형태)
   - 모바일: **가로형 리스트(4.3 디자인)** 1열 — 컴팩트
5. (스크롤 끝에) 광고 배너 한 번 더 (옵션)

### 4.5 학술대회 목록 페이지 (`/conferences`)

**모바일:**
- 헤더 아래 sticky한 검색·필터 바:
  - [검색 input] [필터 ▾] [캘린더/리스트 토글]
- 필터 ▾ 클릭 시 **바텀시트 슬라이드 업**:
  - 진료과 (체크박스 리스트, 상단 검색)
  - 도시 (드롭다운)
  - 기간 (이번달/다음달/3개월/6개월 칩)
  - 하단 [초기화] [N건 보기] 버튼
- 학술대회 가로형 카드 1열 (4.3 디자인) — 무한 스크롤 또는 페이지네이션

**데스크톱:**
- 좌측 sticky 사이드바(280px)에 필터, 우측 콘텐츠
- 카드 그리드 3~4열

### 4.6 학술대회 상세 페이지 (`/conferences/[id]`)

**모바일:**
- 히어로:
  - 배경 그라데이션
  - 빵부스러기(breadcrumb) 작게
  - 배지 (Featured / 등록가능 / D-day / 진료과)
  - 제목 22px / 800
  - 학회명 (링크면 외부 아이콘)
  - 우측 상단: 즐겨찾기 하트 + 공유 버튼 (각 40×40)
- 정보 카드들 (1열로 스택):
  - 일정 정보 (개최일, 장소)
  - 행사 개요 (있을 때)
  - 외부 링크 (KAMS, 학회 홈페이지)
- **하단 고정 sticky bar** (화면 너비 100%, 높이 60px):
  - [♡] [캘린더 추가] [사전등록 하러 가기 →] (메인 CTA가 가장 큼)
  - 페이지 스크롤해도 항상 보임

**데스크톱:** 현재 2단 레이아웃(왼쪽 정보, 오른쪽 sticky 사이드바) 유지

### 4.7 학회 목록 페이지 (`/societies`)

**모바일:**
- 상단 hero: 제목 + 통계 ("총 N개 학회 · M개에서 학술대회 예정")
- 진료과 탭 (가로 스크롤)
- 정렬 토글 (학술대회순/가나다순)
- 학회 카드 그리드 **2열** (현재 1열로 떨어짐 — 2열이 정보 밀도 좋음)
- 카드 한 개:
  - 폭 50% - 5px gap
  - 상단 로고 (40px) + 학회 약자
  - 학회명 (13px, 2줄 ellipsis)
  - 진료과 칩 (10px)
  - 하단 구분선 + "예정 N건" 배지

### 4.8 학회 상세 페이지 (`/societies/[slug]`)

(현재 미구현 또는 기본 형태) — myfair 스타일로 디자인:
- 학회 로고 큰 사이즈 + 학회명 + 진료과 + 홈페이지 링크
- 학회 소개 (있을 때)
- 진행 예정 학술대회 (가로형 카드 리스트)
- 과거 학술대회 (옵션, 컴팩트)

### 4.9 마이페이지 (`/mypage`)

**모바일:**
- 상단: 사용자 이메일 + 로그아웃
- 탭: [즐겨찾기] [알림 설정] [프로필]
- 즐겨찾기: 가로형 카드 리스트 (4.3 디자인)

(이미 `uiux/auth-pages.jsx` 시안대로 구현됨 — 모바일에서도 잘 보이는지 확인만)

### 4.11 푸터

**모바일:**
- 1열 스택

## 5. 출력 형식 (클로드디자인에게)

다음 파일을 만들어줘:

1. **`uiux/mobile-design-v2.jsx`** — 모바일 우선 React/JSX 시안
   - 위 모든 페이지를 한 파일에 섹션별로 (홈/목록/상세/학회목록/학회상세/마이페이지/푸터)
   - CSS variable 사용 (위 토큰 그대로)
   - 360px viewport에서 정상 보이는지 확인
   - 한 파일에 모두 들어가게 — 우리가 보기 편하도록

2. **`uiux/desktop-design-v2.jsx`** (옵션) — 데스크톱 시안
   - 1280px 기준
   - 모바일 시안과 같은 컴포넌트의 데스크톱 버전

3. **각 시안 상단 주석**:
   - 어떤 페이지인지
   - 매핑되는 실제 파일 경로 (예: `→ src/app/page.tsx`, `→ src/components/home/Hero.tsx`)

### JSX 작성 규칙
- React import 안 해도 됨 (시안용)
- 인라인 스타일 OK (실제 토큰 var 사용)
- 더미 데이터(학술대회 5~6건) 포함
- 아이콘은 inline SVG 또는 이모지

## 7. 절대 하지 말 것

- ❌ Tailwind 응답형 유틸리티 (`hidden md:flex`, `md:hidden` 등) 사용 금지 — 운영에서 누락 발생함
- ❌ 인라인 `style={{ padding }}` 같은 응답형 속성 — 미디어 쿼리로 못 덮어씀
- ❌ 데스크톱 카드를 그냥 width:100%로 만든 것을 모바일이라 부르지 말 것
- ❌ 360px에서 가로 스크롤이 생기는 어떤 컴포넌트도 금지
- ❌ 색상 하드코딩 — 무조건 `var(--bm-*)` 사용
